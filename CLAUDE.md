# スケジュール管理PWA

## アプリ概要
iPhoneのSafariでホームに追加して使うPWA。
サーバー不要、LocalStorageでデータ管理するスケジュール帳。
既存のカレンダーアプリと違い、睡眠・移動・食事など1日の全行動を時系列で管理する。

## ファイル構成
- `index.html` — メインアプリ（HTML/CSS/JS単一ファイル）
- `manifest.json` — PWA設定
- `sw.js` — Service Worker（オフライン対応、cache-first + ネットワーク更新）
- `icon.svg` — アプリアイコン（SVG）
- `CLAUDE.md` — このファイル

## 機能
- 1日タイムライン表示（0時〜24時、縦スクロール、1分=1px換算）
- 月カレンダーで日付移動（日付ラベル or 📅ボタンから開く）
- カテゴリ色分け（睡眠/移動/食事/その他、左ボーダー色+背景色で表現）
- 予定の追加・編集・削除（タイムラインの空きタップ or ＋ボタンで新規、予定タップで編集）
- 日をまたぐ予定はその日の範囲にクリップして表示し「継続」マークを付与
- 同時間帯が重なる予定はカラム分割して並べて表示
- 複数日への一括登録（タイトル/カテゴリ/開始時間/終了時間を1回入力し、カレンダーで日付を複数タップ選択して一括登録。各日について `startDate=endDate=その日`、同じ `startTime`/`endTime` のイベントが個別に作成される）
- よく使う予定（お気に入り）— 予定の追加/編集モーダルの☆ボタンでタイトル/カテゴリ/開始時間/終了時間/メモをテンプレートとして登録。メイン画面の「よく使う予定」バーからワンタップでその日（表示中の日付）に予定を追加できる。終了時間が開始時間以下（例: 23:00〜07:00）の場合は日またぎとして翌日を終了日にする。チップの✕で削除
- データエクスポート（JSONファイルダウンロード、`schedule_YYYY-MM-DD.json`）
- データインポート（JSONファイル選択 → 追加/置換を確認ダイアログで選択）
- カレンダーインポート（ショートカットアプリ等のJSON貼り付け → 既存データに追加）

## データ構造
```json
{
  "events": [
    {
      "id": "uuid",
      "startDate": "2026-06-28",
      "startTime": "23:00",
      "endDate": "2026-06-29",
      "endTime": "07:00",
      "title": "睡眠",
      "category": "睡眠",
      "note": ""
    }
  ]
}
```
日付をまたがない予定は `startDate` と `endDate` を同じ日付にする。
LocalStorageキー: `schedule-pwa-data-v1`。

## インポート対応フォーマット
1. 本アプリのエクスポート形式（`startDate`/`startTime`/`endDate`/`endTime`）
2. ISO日時形式（`startDate`/`endDate` が `YYYY-MM-DDTHH:MM:SS` のような1つのフィールドにまとまっている形式）。ショートカットアプリのカレンダー書き出しを想定し、`title`/`name`・`notes`/`note` のどちらのキーでも拾う。カテゴリ情報がない場合は「その他」になる。

## 技術仕様 / 実装メモ
- 単一HTMLファイル（ビルド不要、依存ライブラリなし）
- ダークテーマ固定、日本語UI
- `color-mix()` を使用（iOS Safari 16.2+ で動作。非対応ブラウザでは背景色がカード色にフォールバックするが、カテゴリの左ボーダー色は常に表示される）
- viewport-fit=cover + env(safe-area-inset-*) でノッチ/ホームインジケータに対応
- レイアウトは flexbox（header / legend / timeline を縦積みし、timelineのみスクロール）
- アイコンは `icon.svg` のみ。iOSのホーム画面アイコンとして確実に効かせたい場合は PNG（180x180など）を別途用意して `apple-touch-icon` を差し替えると良い

## GitHub Pages でのホスティング
- 全ファイルをリポジトリのルートに配置
- すべて相対パス（`./`）参照のため、サブディレクトリ配信でもそのまま動作
- Settings → Pages → Branch を指定して公開するだけでOK
